How it works — the two detectors:
Port Scan Detection watches for TCP SYN packets. If a single IP hits 15+ unique ports within 10 seconds, it fires an alert. This catches classic Nmap-style scans. You can tune both numbers at the top of the file (PORT_SCAN_THRESHOLD and PORT_SCAN_WINDOW).
Malicious IP Detection checks every packet's source and destination against a list of known-bad IPs. You populate bad_ips.txt with one IP per line — the file includes links to free public threat feeds you can use for your project.

Tips for your project:

The bad_ips.txt includes RFC 5737 test IPs that are safe to demo with — you can use tools like hping3 or Nmap on your own machine to trigger the port scan alert
Every section is heavily commented explaining the why, which will help when you present it
